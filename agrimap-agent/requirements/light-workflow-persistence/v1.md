## Requirements

1. `light` workflow must not create or retain `.agrimap-agent/tasks/<task-id>/` or task artifacts such as `brief.md`, `checklist.md`, `qa.md`, and `result.md`.
2. `light` workflow must retain durable audit evidence only under `.agrimap-agent/logs/**` and `.agrimap-agent/memory/**`; ignored runtime session/active markers may remain operational state but are not durable evidence.
3. `start`, `checkpoint`, `validate`, `complete`, `close`, and `history` must support light tasks without reading task artifacts.
4. Light validation must prove requester attribution, workflow/execution identity, created and terminal audit events, current/recent memory, claimed files, and verification evidence from logs/memory.
5. `standard` and `regulated` workflow behavior and task artifacts remain unchanged.
6. Existing `.agrimap-agent/tasks/**` history must not be deleted or migrated destructively.
7. Canonical runtime, artifact schema, lifecycle/operation contracts, hooks, tests, documentation, and generated provider mirrors must agree after sync.
8. Existing uncommitted SQL preflight files are outside this requirement and must not be modified or overwritten.
9. Verification must cover new light behavior, unchanged standard/regulated behavior, history attribution, package validation, sync drift, and the full existing test suite.
10. No commit, publish, release, global-cache mutation, or deletion is authorized by this version.
