# Project memory

- Package: Build `agrimap-agent-skills` as an official cross-agent package.
- Architecture: One umbrella skill as source of trust plus thin provider-specific aliases.
- Workspace state directory: `.agrimap-agent/` with task-scoped current memory and logs for multi-person work.
- Legacy decision: `.agm` rules are forbidden; only coding examples were extracted and `.agm` was deleted.
- Package targets: Codex plugin/skill, Claude plugin/skills, and Gemini extension/commands.
- Hooks load context and reinforce checkpoints without adding permission gates.
- Golden SQL files remain byte-for-byte identical; extracted FE/BE examples remain immutable and annotated.
- Missing owner examples: FE libraries, BE libraries, agmbo scheduler, unit-test patterns, and Playground conventions.
- Team identity: resolve the requester per session/task; never keep one shared active-owner or active-task file for the whole project.
- FE discipline: automatically compose phase-aware Front-end Engineer gates with every FE workflow and maintain a vector-ready reusable-artifact index; `/agm-fe-engineer` is a direct entry point.
- Pattern conflicts: raw golden files are evidence only; use `conflict-resolution.md` to separate corrected defects from project-dependent and owner-required choices.
- Delegation: one file and one logical contract have one writer per integration wave; Frontier verifies workspace mode, integrates visible commits/patches/artifacts, dispatches independent read-only QA, and synthesizes evidence.
- QA closure: QA never edits. A failed implementation attempt closes as `qa-failed`; Frontier proposes/discusses a prompt for a new correction task instead of fixing inside the verified task.
- Prompt SoT: one owner-approved prompt is the complete execution source of trust for one task and includes deviation handling plus a structured Result Package.
- Service ownership SoT: use only `.agrimap-agent/knowledge/service-ownership.yaml`; only confirmed evidence may drive hard ownership changes.
- Worktree: Claude supports explicit subagent worktree isolation; Codex managed worktrees are surface-dependent. Detect the actual workspace mode and degrade unknown writers to sequential work.
- Model routing: `fable` is the single Fable 5 label for both reasoning/review and difficult execution; do not use a separate `fable5` entry.
- GitHub: `https://github.com/gasxhermvc/agrimap-agent-skills` is configured as `origin`.
- License: MIT is recommended, but the decision is pending confirmation that preserved golden examples may be publicly relicensed, sanitized, or excluded.
- Backend taxonomy: `agmws` and `agmbo` are the only allowed `backend_profile` values under `target_kind=be-main`; no generic/fallback profile exists.
- Open design: decide the Back-end Engineer discipline separately; proposed independent axes are target kind, host profile, delivery phase, and change kind.
